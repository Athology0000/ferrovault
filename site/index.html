Built like a security tool, not a toy

Every layer makes a deliberate, defensible choice.

🔑

Argon2id + AES-256-GCM

Memory-hard key derivation (64 MiB / t=3 / p=4) feeding authenticated encryption with a fresh nonce on every write.

📦

Tamper-evident format

A custom PVLT binary container whose header is bound as GCM associated data — a parameter downgrade or header edit breaks decryption.

🧹

Zeroized secrets

Master password, derived key, and decrypted entries are wiped from RAM on drop — best-effort, and honest about its limits.

💾

Atomic & locked writes

Temp-file → fsync → atomic rename, under a cross-platform advisory lock. A crash never leaves a half-written vault.

⏱️

TOTP & breach check

Built-in RFC 6238 2FA codes, plus Have I Been Pwned via k-anonymity — only a 5-char hash prefix ever leaves the machine.

🖥️

Sleek TUI & GUI

A polished ratatui terminal UI and a desktop app, switchable with one setting: ferrovault config ui tui|gui.

An honest threat model

Real security tools state what they don't protect, too.

✓ Defends against

  • Vault theft at rest — Argon2id + AES-256-GCM
  • Tampering — GCM tag + header-as-AAD
  • Power loss mid-save — atomic durable writes
  • Concurrent writers — advisory file lock
  • Predictable randomness — OS CSPRNG everywhere

! Does not defend

  • A compromised host — keyloggers / memory scrapers while unlocked
  • Swap & core dumps — zeroization is best-effort
  • A weak master password — Argon2 raises cost, not miracles
  • Clipboard history managers — despite auto-clear
  • Binary tampering — verify what you run

Quick start

Build it, create a vault, launch the UI.

# build from source
$ cargo build --release

# create your vault
$ ferrovault init
$ ferrovault add github --generate

# launch the terminal UI
$ ferrovault ui
# generate a strong password
$ ferrovault gen 32

# 2FA code · breach check
$ ferrovault totp github
$ ferrovault check github

# pick your default UI
$ ferrovault config ui gui
